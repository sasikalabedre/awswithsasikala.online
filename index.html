Capstone 1: Building a Production DevOps Pipeline on AWS

Published: July 2026

In this capstone, I built a complete DevOps pipeline covering 6 layers of modern cloud infrastructure. Starting from a simple Flask application, I containerized it with Docker, provisioned AWS infrastructure with Terraform, deployed to EKS using Helm charts, automated the process with GitHub Actions CI/CD, and finally added enterprise-grade monitoring with Prometheus and Grafana.

Key Learnings:

  • Infrastructure as Code: Terraform enabled repeatable, version-controlled infrastructure deployments
  • Container Orchestration: Kubernetes and Helm simplified multi-pod deployments and rollbacks
  • Automation: GitHub Actions reduced manual deployment time from 30+ mins to under 5 mins
  • Scalability: HPA and Cluster Autoscaler enabled automatic scaling based on demand
  • Observability: Prometheus metrics + Grafana dashboards provided real-time visibility

Takeaway: This project taught me that production systems require more than just writing code — they require automated testing, deployment pipelines, infrastructure planning, and continuous monitoring. The combination of these tools creates a resilient, scalable system that can handle real-world demands.

Capstone 2: GitOps with ArgoCD — Declarative Deployment Automation

Published: July 2026

GitOps is a paradigm shift in how we manage cloud infrastructure. Instead of imperative commands ("do this"), we describe the desired state ("this is what we want"), and tools like ArgoCD automatically make it happen. In this capstone, I implemented a complete GitOps workflow where my GitHub repository became the single source of truth for all infrastructure.

Key Learnings:

  • GitOps Philosophy: Everything in Git = infrastructure, configurations, deployments all version-controlled
  • ArgoCD Auto-Sync: Changed Helm values (replicas 2→5) → ArgoCD detected → 5 pods auto-deployed within 90 seconds
  • Declarative Deployments: No manual `kubectl apply` commands — Helm charts describe desired state
  • CI/CD Integration: GitHub Actions builds images → ECR stores them → ArgoCD automatically deploys
  • Enterprise Monitoring: Prometheus scrapes metrics → Grafana visualizes cluster health in real-time

Takeaway: GitOps represents the future of infrastructure management. By making Git the source of truth, organizations gain better auditability, easier rollbacks, and a clear history of all changes. ArgoCD's ability to continuously reconcile the actual state with the desired state eliminates configuration drift and human error.
